Details of each important topic and what to cover

  • Operating system
    • Operating system → Linux → Ubuntu / Debian
      • File System Hierarchy
        • Root Directory (/): Understanding that the root directory is the top-level directory in the file system hierarchy.
        • Important Directories: Knowledge of key directories such as /bin, /etc, /home, /var, /tmp, etc.
        • Directory Structure: How directories are organized and their purposes.
      • Command-line Operations:
        • Navigating the File System:
          • cd: Change directory.
          • ls: List files and directories.
          • pwd: Print working directory.
        • File and Directory Manipulation:
          • cp: Copy files/directories.
          • mv: Move/rename files/directories.
          • rm: Remove/delete files/directories.
          • mkdir: Create a new directory.
        • File Viewing and Editing:
          • cat, less, more: View file contents.
          • nano, vim, emacs: Text editors for file editing.
        • Searching for Files:
          • find: Search for files in a directory hierarchy.
          • grep: Search for patterns within files.
        • File Permissions:
          • Understanding the concepts of read, write, and execute permissions.
          • Using chmod to change file permissions.
          • Using chown to change file ownership.
        • Wildcard Characters:
          • Using and ? for pattern matching.
      • 3. Users and Permissions:
        • User Management:
          • Creating and deleting users with adduser and deluser.
          • Changing user passwords with passwd.
          • Viewing user information with id and finger.
        • Group Management:
          • Creating and deleting groups with addgroup and delgroup.
          • Adding and removing users from groups with usermod.
        • Sudo and Superuser:
          • Understanding sudo for executing commands with superuser privileges.
          • Editing the sudoers file with visudo.
        • File and Directory Ownership:
          • Understanding the concepts of user and group ownership.
          • Viewing and changing ownership with chown.
    • Operating system → Linux → SUSE Linux:
      • Package Management:
        • SUSE uses the Zypper package manager.
        • Commands like zypper install, zypper remove, zypper update.
      • Configuration Files:
        • Configuration files are often found in /etc/sysconfig/.
        • YAST (Yet Another Setup Tool) is a central configuration tool.
      • Service Management:
        • systemctl is used for managing services.
        • Common services: sshd, network, firewalld.
      • Filesystem Snapshots:
        • Btrfs is often used as the default filesystem, supporting snapshots with snapper.
      • User Management:
        • useradd, userdel, passwd for user management.
        • groupadd, groupdel for group management.
      • Firewall Configuration:
        • firewalld is the default firewall manager.
        • Configurations are usually managed with YaST or firewall-cmd.
    • Operating system → Linux → RHEL (Red Hat Enterprise Linux) / Derivatives:
      • Package Management:
        • RHEL and derivatives use the yum package manager (or dnf in newer versions).
        • Commands like yum install, yum remove, yum update.
      • Configuration Files:
        • Configuration files are often found in /etc/sysconfig/.
        • RHEL uses tools like system-config-* and redhat-config-* for configuration.
      • Service Management:
        • systemctl is used for managing services.
        • Common services: sshd, network, firewalld.
      • SELinux (Security-Enhanced Linux):
        • RHEL often has SELinux enabled for enhanced security.
        • Commands like sestatus for checking SELinux status.
      • User Management:
        • useradd, userdel, passwd for user management.
        • groupadd, groupdel for group management.
      • Firewall Configuration:
        • firewalld or iptables for firewall configuration.
        • Configurations are usually managed with system-config-firewall or firewall-cmd.
      • Subscription Management:
        • RHEL systems require subscription management using tools like subscription-manager.

    • Operating system → Unix → FreeBSD:
      • Package Management:
        • FreeBSD uses the pkg package manager for binary package management.
        • Ports Collection allows compiling applications from source.
      • Configuration Files:
        • Configuration files are typically found in /etc.
        • Tools like sysctl for kernel parameter configuration.
      • Service Management:
        • service and sysrc commands for managing services.
        • rc.conf for system-wide configuration.
      • Filesystem:
        • FreeBSD uses UFS (Unix File System) as the default filesystem.
        • ZFS is also supported and widely used.
      • User Management:
        • pw commands for user and group management.
        • Password policies are defined in /etc/login.conf.
    • Operating system → Unix → OpenBSD:
      • Package Management:
        • OpenBSD uses pkg_add for binary package management.
        • The Ports Collection allows compiling applications from source.
      • Configuration Files:
        • Configuration files are typically found in /etc.
        • OpenBSD focuses on simplicity and security in its configuration.
      • Service Management:
        • rcctl for managing services.
        • The /etc/rc.conf.local file for service configuration.
      • Security Features:
        • OpenBSD is well-known for its security focus, with many security enhancements.
        • Features like W^X (Write XOR Execute) and pledge(2) for process restrictions.
    • Operating system → Unix → NetBSD:
      • Package Management:
        • NetBSD uses pkgsrc (Package Source) for package management.
        • It allows the installation of packages from source.
      • Configuration Files:
        • Configuration files are typically found in /etc.
        • The rc.conf file for system-wide configuration.
      • Service Management:
        • rc.d scripts for managing services.
        • The /etc/rc.conf file for service configuration.
      • Portability:
        • NetBSD is designed for portability and runs on a wide range of hardware platforms.
        • Focus on a clean and portable codebase.
